# Plan: Privacy-Preserving Two-Stage Nostr Route Discovery

## Goal

Implement the first production slice of two-stage route discovery: public Nostr presence stays minimal, route capability flags are public, and exact FIPS/Pollen route descriptors are exchanged only through encrypted on-demand Nostr route request/response messages.

## Tasks

1. Lock route privacy behavior with tests.
   - Files: `test/nostr-mesh-protocol.test.js`, `test/rtc-peer-signaling.test.js`, `test/peer-availability-protocol.test.js` if needed.
   - Verify: targeted tests fail before implementation and pass after.
   - Done: presence omits name/device/platform/descriptors, route requests are NIP-44 encrypted, stale/mismatched responses are rejected, direct route priority remains first.

2. Implement Nostr route descriptor exchange.
   - Files: `public/scripts/nostr-mesh.js`, `public/scripts/network.js`.
   - Action: add session peer id/expiry/capability tags, route request/response event types, pending nonce tracking, descriptor validation, route-room join events, and fallback request hook.
   - Verify: unit tests cover accepted/rejected descriptor flows and route fallback request timing.
   - Done: direct Nostr is tried first; FIPS/Pollen details are requested only after direct failure when capability flags exist.

3. Run repo gates and record evidence.
   - Commands: focused tests, `npm test`, browser e2e if feasible, Docker smoke if feasible, `git diff --check`, changed/full AI-slop scans.
   - Done: summary captures all verification and known gaps.
