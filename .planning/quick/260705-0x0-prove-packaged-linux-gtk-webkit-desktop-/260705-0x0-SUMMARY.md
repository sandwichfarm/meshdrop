# Summary

The packaged Linux GTK/WebKit Desktop Native shell is now automatable through WebKitWebDriver and proves that it loads the
Desktop Native target manifest at runtime.

The native shell does not claim WebRTC or Nostr transfer support yet. Local WebKitGTK exposes no `RTCPeerConnection` or
`RTCDataChannel` API to the packaged shell, so the runtime smoke proves the app gates WebRTC off instead of advertising an
unproven native transfer path.

## Evidence

- `npm run test:desktop-native`
- `node --test test/desktop-package.test.js`
- `node --test test/spa-runtime-config.test.js`
- `npm run test:target-artifacts`

## Remaining Risk

- Desktop Native transfer UAT remains open until the shell uses a native engine that exposes WebRTC data channels.
- Signed installer proof remains open.
