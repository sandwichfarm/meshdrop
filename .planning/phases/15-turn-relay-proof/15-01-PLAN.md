# Phase 15 Plan: TURN Relay Proof

## Goal

Prove that MeshDrop can constrain browser WebRTC file bytes to a configured relay-only TURN path before any overlay route is labeled as WebRTC byte transport.

## Requirements

- TURN-01: local coturn plus MeshDrop RTC config smoke.
- TURN-02: relay-only WebRTC transfer proof with selected ICE candidate type `relay`.
- TURN-03: route proof includes runtime, route, primitive, WebRTC, relay, byte, hash, and fallback fields.
- TURN-04: fail-closed route policy without relay ICE config.
- TURN-05: ADR/docs for relay proof boundary.

- Route proof contract now rejects `webrtc-relay-ice` proofs unless `selectedIceCandidateType` is present and equal to `relay`.

- ADR 0007 records that this is generic TURN relay proof only; route-specific overlay WebRTC labels still need FIPS/Pollen/Tor/I2P/Loki proof.
