# Phase 18 Plan: I2P Byte Transfer Proof

## Goal

Prove an I2P overlay stream route transfers real payload bytes through a Dockerized i2pd `.b32.i2p` endpoint, with hash-matched route proof and no Clearnet fallback claim.

## Scope

- Reuse the existing generic overlay stream transfer storage and HTTP endpoints for I2P.
- Add tests for I2P route proof fields and smoke-script contract.
- Add `npm run test:i2p-stream` that starts a reproducible i2pd HTTP proxy/server tunnel path in Docker and fetches the payload through the `.b32.i2p` route.
- Update ADR/docs and issue #151 with exact I2P proof while keeping Loki blocked.

## Result

Complete on 2026-07-08.

## Risk

- Public I2P bootstrap is slow and environment-dependent; the smoke uses local zero-hop i2pd tunnels to prove the i2pd proxy/server-tunnel data path deterministically.
- Browser WebRTC over I2P is not in scope; this phase proves a non-WebRTC `i2p-http-stream` route.
- Loki remains blocked until an equivalent route-specific harness exists.
