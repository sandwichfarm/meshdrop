# Phase 19 Plan: Loki Byte Transfer Proof

## Goal

Prove that MeshDrop can move bytes through a reproducible local Loki/Lokinet route before claiming Loki transfer support.

## Scope

- Add a Dockerized Lokinet smoke harness.
- Reuse existing generic overlay stream upload/download endpoints.
- Configure MeshDrop with a generated `.loki` stream endpoint.
- Emit and validate route proof for `loki-http-stream`.
- Keep Loki WebRTC and public Lokinet reachability out of scope.

## Acceptance

- `test:loki-stream` exists in `package.json`.
- `scripts/loki-stream-smoke.mjs` builds/starts Lokinet in Docker without mutating host networking.
- Runtime proof transfers a payload through plain `.loki` DNS resolution and reports matching bytes/hash.
- Route proof reports `routeType=loki`, `dataPlanePrimitive=loki-http-stream`, `webRtcUsed=false`, `fallbackUsed=false`, and topology evidence for `.loki`, `127.3.2.1:53`, and `lokitun0`.
- GSD state/docs record phase 19 completion and issue #151 closure criteria.

- Runtime: `npm run test:loki-stream`.
