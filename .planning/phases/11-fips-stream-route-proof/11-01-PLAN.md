# Phase 11 Plan: FIPS Stream Route Proof

## Goal

Transfer private encrypted payload bytes over a FIPS-backed HTTP stream exposed on the sender's FIPS mesh IPv6 address, then emit validated route proof.

## Tasks

1. Add focused red tests for:
   - server FIPS stream upload/download availability, TTL, token, size, and hash behavior;
   - browser FIPS stream descriptor/proof validation and fail-closed cases;
   - network request/download flow emitting route proof;
   - Docker smoke script with two real FIPS daemons.
2. Implement server FIPS stream client and Express routes.
3. Implement browser FIPS stream protocol/controller and load it before `network.js`.
4. Wire `WSPeer` request/accept/download handling and storage route option.
5. Add `npm run test:fips-stream` runtime smoke.
6. Verify focused tests, runtime smoke, broad tests, diff hygiene, and AI-slop gate.

## Verification

- `node --test test/fips-stream-transfer.test.js test/fips-stream-server.test.js`
- `npm run test:fips-stream`
- `npm test`
- `git diff --check`
- `npx --yes aislop scan --changes .`

## Risks

- Browser runtime cannot prove FIPS mesh reachability unless the host can route to the peer's FIPS ULA; Docker smoke proves this inside FIPS-enabled containers.
- This uses HTTP over `fips0`, not a native FSP application API. Native FIPS stream APIs remain future work.
