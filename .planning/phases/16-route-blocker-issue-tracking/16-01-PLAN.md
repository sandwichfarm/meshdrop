# Phase 16 Plan: Route Blocker Issue Tracking

## Objective

Record every remaining route-expansion, release-readback, and deployed-target blocker in live GitHub issues, then update GSD and ADR docs so future milestones start from tracker state instead of stale handoff text.

## Scope

- Verify GitHub issues are enabled and writable.
- Read back existing route blocker issues #151 and #152.
- Create missing blocker issues for GHCR anonymous readback, deployed StartOS/Umbrel UAT, and signed iOS device/share-transfer UAT.
- Update `.planning/PROJECT.md`, `.planning/REQUIREMENTS.md`, `.planning/ROADMAP.md`, `.planning/STATE.md`, and ADR docs with issue links.
- Add a focused docs guard that fails if issue links disappear or stale issue-disabled wording returns.

## Out of Scope

- Implementing Tor/I2P/Loki byte-transfer without a local daemon/proxy.
- Claiming FIPS/Pollen route-specific WebRTC without an overlay-reachable relay endpoint.
- Running deployed StartOS/Umbrel or signed iOS hardware UAT without the required external targets.
- Changing runtime route behavior.
