# Phase 13 Plan: FIPS Instance Relay

## Goal

Make the existing encrypted FIPS stream transfer report through the generic instance-relay descriptor/proof contract, while preserving legacy `fipsStream` request compatibility.

## Requirements

- FIPS-IR-01
- FIPS-IR-02
- FIPS-IR-03
- FIPS-IR-04
- FIPS-IR-05

## Plan

1. Add failing focused tests for FIPS instance-relay descriptor/proof methods, request metadata, recipient proof emission, fail-closed validation, and legacy stream fallback.
2. Add `FipsStreamTransferProtocol` wrappers around `InstanceRelayTransferProtocol` for FIPS route descriptors, proof seeds, request validation, and proof finalization.
3. Attach `fipsInstanceRelay` metadata when a private FIPS request has NIP-44 key delivery and a FIPS room/session, while keeping `fipsStream`.
4. Prefer `fipsInstanceRelay` in recipient download/proof code; fall back to legacy stream validation only when relay metadata is absent.
5. Verify with focused tests, FIPS runtime smoke, broad tests, e2e, Docker, diff check, and changed-code slop scan.

## Acceptance

- `fipsInstanceRelay.descriptor.transportShape === "instance-relay"`.
- Route proof for current FIPS private transfers has `routeType="fips"`, `dataPlanePrimitive="fips-http-stream"`, `webRtcUsed=false`, `instanceRelayed=true`, byte counts, `hashMatched=true`, and `fallbackUsed=false`.
- Malformed relay metadata fails before success.
- `fipsStream`-only legacy requests still emit legacy proof with `instanceRelayed=false`.
