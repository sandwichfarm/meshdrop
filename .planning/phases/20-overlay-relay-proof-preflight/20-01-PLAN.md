# Phase 20 Plan: Overlay Relay Proof Preflight

## Goal

Prevent MeshDrop from accepting FIPS/Pollen/Tor/I2P/Loki WebRTC relay proof unless the proof includes route-specific topology evidence for the overlay relay endpoint.

## Scope

- Keep generic `turn-relay` WebRTC proof behavior unchanged.
- Harden route proof validation for overlay `webrtc-relay-ice` claims.
- Add a fail-closed preflight command for route-specific relay ICE config and topology evidence.
- Update GSD and ADR docs to keep issue #152 as the real byte-transfer blocker.

## Acceptance

- FIPS/Pollen/Tor/I2P/Loki `webrtc-relay-ice` proofs fail without `topologyEvidence`.
- Overlay relay topology evidence must match the route type and name a selected `relayEndpoint`.
- `npm run test:overlay-relay-preflight` exists and fails closed without route-specific config/evidence.
- A configured preflight run reports `provenTransfer=false` and does not claim route bytes moved.
- Issue #152 remains open until a real overlay-reachable relay endpoint carries browser transfer bytes.
