# Phase 12 Plan: Generic Instance Relay

## Goal

Extract the proven Pollen instance-relay descriptor and proof semantics into a reusable generic protocol without changing live Pollen transfer behavior or claiming new network support.

## Tasks

1. Add focused red tests for a generic instance-relay protocol:
   - descriptor construction and validation;
   - proof seed and proof finalization;
   - fail-closed owner/session/expiry/primitive/fallback/WebRTC cases.
2. Implement `InstanceRelayTransferProtocol` as a browser-loaded script after `route-contract.js`.
3. Refactor `PollenTransferProtocol` instance-relay methods to delegate descriptor/proof logic to the generic protocol while preserving the existing `pollenInstanceRelay` request shape.
4. Add ADR 0006 documenting the reusable relay boundary and future transport rules.
5. Verify focused tests, broad tests, browser smoke if script load order changes, diff hygiene, and changed-code AI-slop.

## Verification

- `node --test test/instance-relay-transfer.test.js test/pollen-instance-relay.test.js test/pollen-transfer-protocol.test.js test/route-contract.test.js`
- `npm test`
- `npm run test:e2e`
- `git diff --check`
- `npx --yes aislop scan --changes .`

## Risks

- This must not make Pollen behavior look new to users; it is a refactor under the existing public request/proof shape.
- A generic helper used once can become slop unless it removes real duplicate semantics and is shaped for the immediately next FIPS relay slice.
