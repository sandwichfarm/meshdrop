# Phase 21 Plan: Instance-Backed FIPS/Pollen ICE Bridge

## Scope

Small vertical slice from server capability descriptor to trusted WOT route metadata to bridge-constrained peer connection setup.

## Steps

1. Add instance ICE bridge descriptor normalization on the server while preserving existing TURN-style route config.
2. Add client ICE bridge descriptor normalization that accepts trusted route metadata and preserves source/topology evidence.
3. Include ICE bridge descriptors in FIPS/Pollen WOT route descriptors and preserve them through Nostr route response normalization.
4. Store selected route metadata by room and use it in route blocking and `RTCPeerConnection` config.
5. Update status/copy tests so labels separate WOT discovery/signaling, instance ICE bridge, and transfer primitives.
6. Add focused regressions and an executable instance ICE bridge smoke.
7. Run focused tests, smoke, `npm test`, `git diff --check`, and changed-code AI-slop scan.

## Risk

Browser WebRTC still requires TURN/TURNS-shaped ICE server entries. This slice makes those entries instance-backed and route-scoped, but it does not implement the bridge daemon itself.
